# 백트래킹 냄새 찾기 연습

> 문제 설명만 보고 "이거 백트래킹이다"를 떠올릴 수 있는가?

---

## 1. N과 M (1) (BOJ 15649)

### <문제>
자연수 N과 M이 주어진다 (1 ≤ M ≤ N ≤ 8).
1부터 N까지의 자연수 중에서 중복 없이 M개를 골라 만든 수열을 모두 출력하라.
사전순으로 출력한다.

### <정답>
**냄새:** `중복 없이 M개를 골라` + `만든 수열을 모두 출력` → **"가능한 조합을 전부 나열하라"**

"모두 출력하라"가 핵심이다. 최솟값도, 개수도 아닌, **가능한 경우를 전부 만들어라**는 뜻이다.
1~N 중 하나를 고르고 → 남은 것 중 하나를 고르고 → … M개를 채울 때까지 반복한다.
M개를 채우면 출력하고, 한 칸 되돌아가서 다른 선택지를 시도한다.
이 "고르고 → 더 들어가고 → 막히면 되돌아감" 구조가 백트래킹의 정의 그 자체다.
**"가능한 순열/조합을 전부 나열 = 백트래킹"**

---

## 2. 연산자 끼워넣기 (BOJ 14888)

### <문제>
N개의 수로 이루어진 수열이 있고, 수 사이에 끼워넣을 N-1개의 연산자(+, -, ×, ÷)가 주어진다.
수의 순서는 바꿀 수 없고, 연산자의 배치만 바꿀 수 있다.
연산자 우선순위를 무시하고 앞에서부터 계산할 때, 결과의 최댓값과 최솟값을 구하라.

### <정답>
**냄새:** `연산자의 배치만 바꿀 수 있다` + `최댓값과 최솟값` → **"모든 배치를 시도해서 극값을 찾아라"**

연산자 N-1개의 모든 배치 순서를 시도해야 한다. 첫 번째 자리에 +를 넣어보고 → 두 번째 자리에 -를 넣어보고 → … 끝까지 채우면 결과를 계산한다.
그다음 한 칸 되돌아가서 다른 연산자를 넣어본다. 남은 연산자 개수를 관리하며 모든 순서를 탐색하는 구조다.
N이 최대 11이므로 최악 10! = 약 360만인데, 같은 연산자끼리는 중복이니 실제로는 훨씬 적다.
**"제한된 개수의 선택지를 매 자리마다 골라 모든 경우 탐색 = 백트래킹"**

---

## 3. 치킨 배달 (BOJ 15686)

### <문제>
N×N 도시에 집과 치킨집이 있다.
치킨 거리 = 집에서 가장 가까운 치킨집까지의 맨해튼 거리.
도시의 치킨 거리 = 모든 집의 치킨 거리의 합.
치킨집 중 최대 M개만 남기고 나머지를 폐업시킬 때, 도시의 치킨 거리의 최솟값을 구하라.

### <정답>
**냄새:** `치킨집 중 M개를 고른다` + `최솟값` → **"여러 개 중 M개를 골라 모든 조합을 시도"**

치킨집이 최대 13개, M이 최대 13이므로 조합 수(₁₃Cₘ)는 충분히 탐색 가능하다.
치킨집 목록에서 M개를 고르는 모든 조합을 만들고, 각 조합마다 도시의 치킨 거리를 계산해서 최솟값을 갱신한다.
"M개 고르기"를 구현할 때: 첫 번째 치킨집을 포함할지 말지 → 두 번째를 포함할지 말지 → … M개를 채우면 계산, 안 되면 되돌아감.
이 조합 생성 과정 자체가 백트래킹이다.
**"N개 중 M개 조합 선택 + 최적값 탐색 = 백트래킹"**

## 4. 부분수열의 합 (BOJ 1182)

### <문제>
N개의 정수로 이루어진 수열이 주어진다 (N ≤ 20).
크기가 양수인 부분수열(원소를 1개 이상 선택) 중
원소의 합이 S가 되는 경우의 수를 구하라.

### <정답>
**냄새:** `부분수열` + `합이 S가 되는 경우의 수` → **"각 원소를 넣거나 빼며 모든 경우를 탐색"**

각 원소마다 "포함한다 / 포함하지 않는다" 두 가지 선택이 있다. N개 원소에 대해 이 선택을 반복하면 2^N가지 부분수열이 만들어진다.
N ≤ 20이므로 2^20 = 약 100만, 완전 탐색이 가능하다.
첫 번째 원소를 포함 → 두 번째 원소를 포함 → … 끝까지 가서 합을 확인 → 되돌아가서 "포함하지 않음"을 시도.
매 단계에서 이진 분기를 하고, 끝까지 갔다가 되돌아오는 전형적인 백트래킹 구조다.
**"각 원소를 선택/비선택하며 전수 탐색 = 백트래킹"**

## 정리: 백트래킹 냄새 체크리스트

| 냄새 | 핵심 질문 |
|------|----------|
| 모든 경우를 나열 | "가능한 순열/조합을 전부 출력?" |
| N개 중 M개 선택 | "여러 개 중 일부를 골라 조건 확인?" |
| 포함/비포함 이진 분기 | "각 원소를 넣거나 빼는 2^N 탐색?" |
| 모든 배치에서 극값 | "배치를 전부 시도해서 최대/최소?" |
| **공통점** | **경우의 수가 작고(N ≤ ~20), 모든 경우를 시도해서 답을 찾음** |
