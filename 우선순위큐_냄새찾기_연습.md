# 우선순위큐 냄새 찾기 연습

> 문제 설명만 보고 "이거 우선순위큐다"를 떠올릴 수 있는가?

---

## 1. 카드 합체 놀이 (BOJ 15903)

### <문제>
자연수가 쓰인 카드 n장이 있다.
카드 2장을 골라 두 수의 합을 구한 뒤, 그 합을 두 카드 모두에 덮어쓴다.
이 합체를 m번 수행한 뒤, 모든 카드에 쓰인 수의 합(=점수)을 가장 작게 만들어라.

### <정답>
**냄새:** `카드 2장을 골라 합체` + `점수를 가장 작게` → **"매번 가장 작은 두 개를 골라야 최소"**

합체하면 두 카드 모두 합으로 바뀌므로, 작은 수끼리 합쳐야 전체 합이 최소가 된다.
매번 "지금 가장 작은 두 장"을 빠르게 꺼내야 하고, 합체 결과(새 값)를 다시 후보에 넣어야 한다.
이 "꺼내고 → 처리하고 → 다시 넣기"를 반복하면서 매번 최솟값을 유지하는 구조가 최소 힙이다.
**"매번 가장 작은 것을 골라 합치고 다시 넣기 = 우선순위큐"**

---

## 2. 강의실 (BOJ 1374)

### <문제>
N개의 강의가 있고, 각 강의의 시작 시간과 끝나는 시간이 주어진다.
한 강의실에서 동시에 2개 이상의 강의를 진행할 수 없다.
(종료 시간 = 시작 시간은 허용)
모든 강의를 진행하기 위해 필요한 최소 강의실 수를 구하라.

### <정답>
**냄새:** `동시에 진행 불가` + `최소 강의실 수` → **"겹치는 구간이 최대 몇 개인가"**

강의를 시작 시간 순으로 정렬하고, 새 강의가 시작될 때 "가장 빨리 끝나는 강의실"이 비었는지 확인한다.
비었으면 그 강의실을 재사용하고, 안 비었으면 강의실을 하나 추가한다.
"가장 빨리 끝나는 강의실"을 매번 빠르게 꺼내려면? → 종료 시간을 최소 힙에 넣으면 된다.
새 강의 시작 시간 ≥ 힙의 top(가장 빠른 종료 시간)이면 poll하고 새 종료 시간을 넣고, 아니면 그냥 넣는다.
힙의 크기 = 현재 사용 중인 강의실 수.
**"가장 빨리 끝나는 것을 매번 확인 = 우선순위큐"**

## 3. 최소 회의실 개수 (BOJ 19598)

### <문제>
N개의 회의가 있고, 각 회의의 시작 시간과 끝나는 시간이 주어진다.
한 회의실에서 동시에 두 개 이상의 회의를 진행할 수 없고, 회의는 중단할 수 없다.
끝나는 시간과 시작 시간이 같으면 같은 회의실을 쓸 수 있다.
모든 회의를 진행할 수 있는 최소 회의실 개수를 구하라.

### <정답>
**냄새:** `동시에 진행 불가` + `최소 회의실 개수` → **"겹치는 구간이 최대 몇 개인가"** (1374번과 동일 패턴)

1374번(강의실)과 본질적으로 같은 문제다.
회의를 시작 시간 순으로 정렬하고, 최소 힙에 종료 시간을 관리한다.
새 회의가 시작될 때 힙 top(가장 빨리 끝나는 회의실)의 종료 시간 ≤ 새 시작 시간이면 재사용(poll 후 add), 아니면 회의실 추가(add만).
힙의 최대 크기가 답이다.
**"스케줄 겹침 관리 + 가장 빨리 끝나는 것 확인 = 우선순위큐"**

---

## 4. 아이들과 선물 상자 (BOJ 23757)

### <문제>
선물 상자가 여러 개 있고, 각 상자에 선물이 일정 개수 들어있다.
아이들이 순서대로 와서 가장 선물이 많은 상자에서 원하는 만큼 선물을 가져간다.
모든 아이들이 원하는 만큼 선물을 받을 수 있는지 판별하라.

### <정답>
**냄새:** `가장 선물이 많은 상자에서` + `순서대로 가져감` → **"매번 가장 큰 것을 꺼내야 한다"**

아이가 올 때마다 "현재 선물이 가장 많은 상자"를 찾아야 한다. 선물을 가져가면 상자의 수가 줄어들고, 줄어든 상자를 다시 후보에 넣어야 한다.
"가장 큰 값을 매번 빠르게 꺼내기 → 처리 → 다시 넣기" = 최대 힙이다.
최대 힙의 top에서 아이가 원하는 수를 빼고, 남은 수를 다시 힙에 넣는다. top이 아이가 원하는 수보다 작으면 불가능.
**"매번 가장 큰 것을 꺼내서 처리하고 다시 넣기 = 우선순위큐(최대 힙)"**

## 5. 카드 정렬하기 (BOJ 1715)

### <문제>
정렬된 카드 묶음이 N개 있다.
두 묶음을 합칠 때 각 묶음의 크기가 A, B이면 A+B번의 비교가 필요하다.
N개의 묶음을 하나로 합칠 때, 필요한 최소 비교 횟수를 구하라.

### <정답>
**냄새:** `두 묶음을 골라 합친다` + `비교 횟수를 최소` + `합친 결과가 다시 합치기 후보` → **"매번 가장 작은 두 개를 골라야 최소"**

작은 묶음끼리 먼저 합쳐야 나중에 큰 묶음이 반복 합산되는 횟수가 줄어든다.
매번 가장 작은 두 묶음을 꺼내고(poll 2번), 합친 뒤 다시 넣는다(add). 이걸 묶음이 1개가 될 때까지 반복한다.
합친 결과가 다시 후보가 되므로, 동적으로 삽입·삭제하며 최솟값을 유지하는 최소 힙이 필요하다.
**"꺼내고 → 합치고 → 다시 넣기를 반복하며 매번 최솟값 = 우선순위큐"**

## 정리: 우선순위큐 냄새 체크리스트

| 냄새 | 핵심 질문 |
|------|----------|
| 매번 최솟값/최댓값 선택 | "매번 가장 작은(큰) 것을 골라야 하나?" |
| 꺼내고 → 처리 → 다시 넣기 | "처리 결과가 다시 후보가 되나?" |
| 합치기 반복 | "두 개를 합치고, 합친 것으로 또 합치나?" |
| 스케줄 겹침 관리 | "가장 빨리 끝나는 것을 매번 확인하나?" |
| **공통점** | **동적으로 변하는 후보 중에서 매번 극값을 빠르게 꺼내야 함** |
