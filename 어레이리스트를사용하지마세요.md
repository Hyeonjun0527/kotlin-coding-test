맞아. **“처음부터 y개 칸을 기본값으로 채운 2차원”** 목적이면 `ArrayList`는 함정이 많아. (`ArrayList(yCnt)`는 **capacity만** 잡고 **size=0**이라 기본값 채우기가 안 됨)

근데 정확히는:

* `ArrayList`도 **기본값으로 미리 채울 수는 있어**
  다만 **생성자에서 `{}`로 못 하고**, `add`를 반복하거나 `Collections.nCopies` 같은 걸 써야 해서 번거로움.

예시 (가능은 함):

```kotlin
val row = ArrayList<Int>(yCnt).apply { repeat(yCnt) { add(0) } } // size=yCnt로 채움
```

그래서 코테/격자에서는 보통 이렇게 고정해:

### 1) 정석(빠르고 깔끔)

```kotlin
val dist = Array(xCnt) { IntArray(yCnt) } // 0으로 채워짐
```

### 2) “리스트로 size 채우고 싶다”면

```kotlin
val dist = MutableList(xCnt) { MutableList(yCnt) { 0 } } // size가 바로 yCnt
```

한 줄 결론:

* **고정 격자 = `Array { IntArray }`**
* **가변 길이 = `ArrayList`/`MutableList` + add**
* **초기값 채운 리스트 격자 = `MutableList(n){...}`가 더 편함**
