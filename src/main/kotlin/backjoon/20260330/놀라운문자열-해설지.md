# 놀라운 문자열 해설지

## 처음 어떤 신호를 보고 이 알고리즘을 떠올려야 했는가
정의가 `D-쌍` 기준으로 나뉘어 있다는 게 핵심 신호다. 그러면 문제는 "각 거리 D마다 쌍이 모두 유일한지 검사"로 분해된다. 거리 하나를 고정하고 한 칸씩 밀면서 쌍을 set에 넣는 구조가 바로 나온다.

## 내가 왜 그걸 못 떠올렸는가
문자열 문제를 보면 substring, 정렬, 빈도수부터 떠올리는 습관이 있다. 그런데 이 문제는 문자 빈도 자체는 중요하지 않고, "간격이 같은 두 문자 조합"의 중복 여부만 본다. 기준 축을 D로 못 바꾸면 계속 헛돈다.

## 틀린 방향은 왜 틀렸는가
모든 쌍을 하나의 전역 set에 넣는 건 틀리다. 이 문제는 거리마다 독립적으로 유일성을 검사해야 하므로, 같은 문자쌍이 다른 거리에서 다시 나와도 된다. 거리 축을 섞는 순간 문제 정의를 어기게 된다.

## 다음에 비슷한 문제가 나오면 어디를 먼저 볼 것인가
문제 정의에 `각 k마다`, `각 거리마다`, `각 길이마다` 같은 표현이 있으면 바깥 루프 후보로 바로 올린다. 전체를 한 번에 보려 하지 말고, 정의에 나온 축대로 분해할 수 있는지 먼저 본다.
