# 오목 알고리즘 상태 전이 그래프

한 다이어그램 안에서 `S`(흐름)와 `V`(상태)를 분리해서 본다.

## 1) 통합 다이어그램 (S+V)

```mermaid
graph TB
  subgraph STATE["V 상태 나열"]
    direction LR
    V0["V0 board(list)"]
    V1["V1 dr/dc"]
    V2["V2 현재 돌 값"]
    V3["V3 연속 개수"]
    V4["V4 winner"]
    V5["V5 승리 좌표 sol"]
    V6["V6 isRange"]
  end

  subgraph FLOW["S 실행 흐름"]
    direction TB
    S0["S0 보드 입력<br/>19x19 로드 : V0"]
    S1["S1 모든 칸 순회<br/>시작점 후보 탐색 : V0"]
    S2["S2 돌 존재 검사<br/>0이 아닌가 : V0,V2"]
    S3["S3 방향 후보 순회<br/>8방향 검사 : V1"]
    S4["S4 시작점 검증<br/>반대칸 동일색이면 스킵 : V0,V1,V6"]
    S5["S5 연속 개수 세기<br/>같은 색 진행 : V0,V1,V3,V5"]
    S6["S6 정확히 5개 검사<br/>count==5 ? : V3"]
    S7["S7 다음칸 동일색 검사<br/>장목 배제 : V0,V1,V6"]
    S8["S8 승자 확정 후 종료<br/>winner/좌표 기록 : V4,V5"]
    S9["S9 계속 스캔<br/>다음 후보로 진행 : V0"]
    S10["S10 끝까지 없음 출력<br/>0 출력 : V4"]

    S0 --> S1 --> S2
    S2 -- 아니오 --> S9 --> S1
    S2 -- 예 --> S3 --> S4
    S4 -- 스킵 --> S3
    S4 -- 통과 --> S5 --> S6
    S6 -- 아니오 --> S3
    S6 -- 예 --> S7
    S7 -- 아니오 --> S8
    S7 -- 예 --> S3
    S1 -- 전체완료 --> S10
  end
```

## 2) V 갱신 규칙 (S 단계 기준)

- `S5`: `V3,V5` 연속 라인 갱신
- `S8`: `V4,V5` 승자/좌표 확정
- `S10`: `V4` 기반으로 0 출력

## 직관 요약

흐름은 `후보 스캔 -> 5목 검증 -> 장목 배제`이고,
상태 관리는 `V0~V6` 정의표와 갱신 규칙표로 추적한다.
