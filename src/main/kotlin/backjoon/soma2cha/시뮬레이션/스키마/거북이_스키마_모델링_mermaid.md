# 거북이 알고리즘 상태 전이 그래프

한 다이어그램 안에서 `S`(흐름)와 `V`(상태)를 분리해서 본다.

## 1) 통합 다이어그램 (S+V)

```mermaid
graph TB
  subgraph STATE["V 상태 나열"]
    direction LR
    V0["V0 x"]
    V1["V1 y"]
    V2["V2 dir"]
    V3["V3 minX/maxX/minY/maxY"]
    V4["V4 dx/dy"]
    V5["V5 결과버퍼 sb"]
    V6["V6 현재 명령열 orderList"]
    V7["V7 테스트케이스 반복 상태"]
  end

  subgraph FLOW["S 실행 흐름"]
    direction TB
    S0["S0 테스트케이스 시작<br/>x=0, y=0, dir=0, min/max 초기화 : V0,V1,V2,V3,V7"]
    S1["S1 명령 1개 읽기<br/>현재 문자 확인 : V6"]
    S2["S2 명령 종류 분기<br/>L/R/F/B 분기 : V2,V6"]
    S3["S3 L 회전<br/>dir=(dir+3)%4 : V2"]
    S4["S4 R 회전<br/>dir=(dir+1)%4 : V2"]
    S5["S5 F/B 이동<br/>좌표 이동 : V0,V1,V2,V4"]
    S6["S6 경계 갱신<br/>min/max 갱신 : V0,V1,V3"]
    S7["S7 명령 남음 검사<br/>다음 문자 있는지 확인 : V6"]
    S8["S8 넓이 계산/기록<br/>width*height 저장 : V3,V5"]
    S9["S9 테스트케이스 남음 검사<br/>다음 케이스 여부 : V7"]
    S10["S10 전체 출력<br/>sb 출력 : V5"]

    S0 --> S1 --> S2
    S2 -- L --> S3 --> S7
    S2 -- R --> S4 --> S7
    S2 -- F/B --> S5 --> S6 --> S7
    S7 -- 예 --> S1
    S7 -- 아니오 --> S8 --> S9
    S9 -- 예 --> S0
    S9 -- 아니오 --> S10
  end
```

## 2) V 갱신 규칙 (S 단계 기준)

- `S0`: `V0,V1,V2,V3,V7` 초기화
- `S3,S4`: `V2` 회전 갱신
- `S5`: `V0,V1` 이동 갱신
- `S6`: `V3` 경계 갱신
- `S8`: `V5`에 케이스 결과 추가
- `S10`: `V5` 최종 출력

## 직관 요약

흐름은 `명령 재생 -> 좌표/경계 갱신 -> 넓이 기록`을 케이스마다 반복하고,
상태 관리는 `V0~V7` 정의표와 갱신 규칙표로 추적한다.
