# 주차장 알고리즘 상태 전이 그래프

한 다이어그램 안에서 `S`(흐름)와 `V`(상태)를 분리해서 본다.

## 1) 통합 다이어그램 (S+V)

```mermaid
graph TB
  subgraph STATE["V 상태 나열"]
    direction LR
    V0["V0 빈칸PQ pq"]
    V1["V1 대기큐 dq"]
    V2["V2 차->칸 매핑"]
    V3["V3 주차칸 단가"]
    V4["V4 차량 무게"]
    V5["V5 수입"]
    V6["V6 이벤트목록 actionList"]
    V7["V7 현재 차량번호"]
  end

  subgraph FLOW["S 실행 흐름"]
    direction TB
    S0["S0 초기상태<br/>pq/dq/매핑/수입 초기화 : V0,V1,V2,V5"]
    S1["S1 이벤트 1개 읽기<br/>현재 이벤트 로드 : V6,V7"]
    S2["S2 입차 이벤트 검사<br/>x>0 ? : V7"]
    S21["S21 빈칸 존재 검사<br/>pq 비었는지 확인 : V0"]
    S3["S3 즉시 배정<br/>칸 할당 : V0,V2,V7"]
    S4["S4 대기열 추가<br/>차량 enqueue : V1,V7"]
    S5["S5 출차 처리<br/>정산 + 칸 반납 : V0,V2,V3,V4,V5,V7"]
    S6["S6 대기차 검사<br/>dq 비었는지 확인 : V1"]
    S7["S7 대기차 즉시 배정<br/>dequeue 후 배정 : V0,V1,V2"]
    S8["S8 이벤트 남음 검사<br/>다음 이벤트 여부 : V6"]
    S9["S9 최종 수입 출력<br/>print 수입 : V5"]

    S0 --> S1 --> S2
    S2 -- 예 --> S21
    S21 -- 예 --> S3 --> S8
    S21 -- 아니오 --> S4 --> S8
    S2 -- 아니오 --> S5 --> S6
    S6 -- 예 --> S7 --> S8
    S6 -- 아니오 --> S8
    S8 -- 예 --> S1
    S8 -- 아니오 --> S9
  end
```

## 2) V 갱신 규칙 (S 단계 기준)

- `S3`: `V0,V2` 즉시 배정 반영
- `S4`: `V1` 대기열 추가
- `S5`: `V5` 정산 누적, `V0,V2` 출차 반영
- `S7`: `V0,V1,V2` 대기 해소 배정

## 직관 요약

흐름은 이벤트를 순서대로 `입차/출차` 분기 처리하고,
상태 관리는 `V0~V7` 정의표와 갱신 규칙표로 추적한다.
