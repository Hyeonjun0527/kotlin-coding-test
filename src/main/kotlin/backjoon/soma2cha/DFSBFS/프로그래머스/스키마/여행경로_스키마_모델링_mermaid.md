# 여행경로 스키마 모델링

코드의 `[S*]`, `[V*]` 집합과 실행 순서를 맞춘 버전이다.

```mermaid
graph TB
  A1["S0 초기화"]
  A2["S1 인접목록맵 구성"]
  A3["S2 도착목록 내림차순 정렬"]
  A4["S3 시작 공항 스택 삽입"]
  A5["S4 반복형 Hierholzer"]
  C1["C1 경로스택이 비었는가"]
  C2["C2 현재공항에 남은 도착목록이 있는가"]
  A6["S5 역순경로 뒤집기"]
  A7["S_END 결과 반환"]

  B1["V0 티켓목록"]
  B2["V1 인접목록맵"]
  B3["V2 출발공항"]
  B4["V3 도착목록"]
  B5["V4 경로스택"]
  B6["V5 역순경로"]
  B7["V6 현재공항"]
  B8["V7 다음공항"]
  B9["V8 정답"]

  A1 --> A2
  A2 --> A3
  A3 --> A4
  A4 --> A5
  A5 --> C1
  C1 -- "YES" --> A6
  C1 -- "NO" --> C2
  C2 -- "YES" --> A5
  C2 -- "NO" --> A5
  A6 --> A7

  A1 --> B1
  A1 --> B2
  A1 --> B5
  A1 --> B6
  A2 --> B1
  A2 --> B2
  A2 --> B3
  A3 --> B2
  A3 --> B4
  A4 --> B5
  A5 --> B2
  A5 --> B4
  A5 --> B5
  A5 --> B6
  A5 --> B7
  A5 --> B8
  A6 --> B6
  A6 --> B9
```

## S/V 싱크 체크 결과
- S 집합: `S0,S1,S2,S3,S4,S5,S_END` 코드/스키마 일치
- V 집합: `V0,V1,V2,V3,V4,V5,V6,V7,V8` 코드/스키마 일치
