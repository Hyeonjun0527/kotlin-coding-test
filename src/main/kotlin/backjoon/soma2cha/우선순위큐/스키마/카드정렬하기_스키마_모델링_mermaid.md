# 카드정렬하기 스키마 모델링

```mermaid
graph TB
  A1["S0 상태 초기화"]
  A2["S1 묶음수가 1 이하인가"]
  A3["S2 힙 크기가 1보다 큰가"]
  A4["S3 최소 2묶음 추출"]
  A5["S4 합치고 비용 누적 후 힙 삽입"]
  A6["S_END 총비용 반환"]

  B1["V0 묶음수"]
  B2["V1 카드힙"]
  B3["V2 총비용"]
  B4["V3 묶음A"]
  B5["V4 묶음B"]
  B6["V5 합친묶음"]

  A1 --> A2
  A2 -- "YES" --> A6
  A2 -- "NO" --> A3
  A3 -- "YES" --> A4
  A3 -- "NO" --> A6
  A4 --> A5
  A5 --> A3

  A1 --> B1
  A1 --> B2
  A1 --> B3
  A4 --> B4
  A4 --> B5
  A5 --> B6
```

## S/V 싱크 체크 결과
- S 집합: `S0,S1,S2,S3,S4,S_END` 코드/스키마 일치
- V 집합: `V0,V1,V2,V3,V4,V5` 코드/스키마 일치
