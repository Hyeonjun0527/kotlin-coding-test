# 2026-01-19 DP 게임 이론 공부 후기

## DP의 본질

DP 아이디어는 사실 뭐?
"**상태(state) 만들기**에서 시작함"

게임 문제 보면 제일 먼저?
"지금 상황을 숫자 하나로 요약할 수 있나?"

## DP 설계 5단계 (게임 문제)

### 1) 상태가 뭐지?
"문제를 숫자로 표현"
```
예: 남은 돌 개수 i
```

### 2) dp에 뭘 저장하지?
"이 상태에서 현재 차례가 이기는지"
```
dp[i] = true/false
```

### 3) 다음 상태는?
"어떤 선택이 가능한가"
```
i → i-1, i-3
(1개 또는 3개 가져가기)
```

### 4) 승리 조건은?
"상대를 지는 상태로 보내는 선택이 있으면 승"
```kotlin
if (!dp[i-1] || !dp[i-3]) {
    dp[i] = true  // 둘 중 하나라도 false면 승리
}
```

### 5) 초기값은?
"자명한 상태부터 채우기"
```
dp[0] = false  // 돌이 0개면 가져갈 게 없으니 짐
```

## 게임 DP 저장값 패턴

가장 흔한 저장값?
"현재 차례인 사람이 이길 수 있냐?"
```
돌이 i개 남았을 때 지금 차례인 사람이 이긴다
  => dp[i] = true

돌이 i개 남았을 때 지금 차례인 사람이 진다
  => dp[i] = false
```
