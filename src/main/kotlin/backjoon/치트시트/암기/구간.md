끝 미포함 구간 `[시작, 끝)`가 편한 이유는,  
이 표현이 사람에게 “몇 개가 들어 있나”, “어디서 끊기나”, “다음 구간은 어디서 시작하나”를 훨씬 단순하게 보여주기 때문이다.  
이 방식에서는 구간의 길이가 항상 `끝 - 시작`이므로, 길이를 계산할 때 따로 `+1`을 붙일 필요가 없다.  
또 비어 있는 구간도 `[x, x)`처럼 자연스럽게 표현할 수 있어서, 재귀 종료 조건이나 반복문 범위를 작성할 때 매우 깔끔해진다.  
두 구간이 맞닿아 있을 때도 `[a, b)`와 `[b, c)`처럼 같은 숫자를 경계로 바로 이어 붙일 수 있어서, 겹침이나 누락이 없는지 확인하기 쉽다.  
이 특징은 특히 반으로 나누기, 3등분하기, 병합하기 같은 분할 정복 문제에서 큰 장점이 된다.  
끝 포함 방식은 처음 보면 직관적인 것 같지만, 실제 구현으로 들어가면 `+1`, `-1` 조정이 계속 필요해서 실수하기 쉽다.  
반면 끝 미포함 방식은 코틀린의 `until`과도 잘 맞기 때문에, 머릿속 구간 표현과 코드 표현이 거의 일치한다.  
결국 `[시작, 끝)`는 더 똑똑한 표현이라기보다, **실수할 틈을 줄여 주는 표현**이라고 보는 게 맞다.  
그래서 한 문제 안에서는 이 표현 하나로 통일하는 습관이 매우 중요하다.

| 항목                  | `[s, e)`                 | `[s, e]`                  |
| ------------------- | ------------------------ | ------------------------- |
| `e`의 의미             | 끝 경계                     | 마지막 원소 인덱스                |
| 실제 처리 범위            | `s ~ e-1`                | `s ~ e`                   |
| 길이                  | `e - s`                  | `e - s + 1`               |
| 빈 구간 표현             | 가능 (`s == e`)            | 애매함                       |
| 중간값 `mid` 구하는 식     | `(s + e) / 2`            | `(s + e) / 2`             |
| `mid`의 의미           | 오른쪽 시작점                  | 왼쪽 마지막 원소                 |
| `mid` 기준 왼쪽 구간      | `[s, mid)`               | `[s, mid]`                |
| `mid` 기준 오른쪽 구간     | `[mid, e)`               | `[mid+1, e]`              |
| 왼쪽 길이               | `mid - s`                | `mid - s + 1`             |
| 오른쪽 길이              | `e - mid`                | `e - mid`                 |
| 홀수 길이일 때 가운데 원소 인덱스 | `(s + e - 1) / 2`        | `(s + e) / 2`             |
| 짝수 길이일 때 가운데        | 가운데 하나 없음                | 가운데 하나 없음                 |
| 바로 붙는 두 구간          | `[a,b)`와 `[b,c)`         | `[a,b]`와 `[b+1,c]`        |
| 재귀 분할               | `+1/-1` 거의 없음            | 경계마다 `+1/-1` 고민           |
| 길이 0 조건             | `s == e`                 | 보통 잘 안 씀 / `s > e`를 별도로 봄 |
| 길이 1 조건             | `e - s == 1`             | `s == e`                  |
| 길이 0 또는 1 종료 조건     | `e - s <= 1`             | `s >= e`                  |
| 병합정렬 종료 조건 추천       | `if (e - s <= 1) return` | `if (s >= e) return`      |
| 한 줄 암기              | mid는 오른쪽 시작              | mid는 왼쪽 끝                 |





- **칸토어, 분할정복, 문자열 구간 자르기** → `(시작, 길이)` 좋음
- **일반 배열 구간, 투포인터, 병합 구간** → `[시작, 끝)` 좋음
- **끝 포함 `[s,e]`** 는 문제 원문이 그 식으로 주어졌을 때만 잠깐 쓰고, 코드에서는 가능하면 바꾸는 편이 안전함

만약 n이 짝수면 가운데 원소 인덱스는 존재하지 않는다.
그리고, 가운데 원소 인덱스가 존재한다면(=n이 홀수면)
가운데 원소 인덱스는 언제나 (시작원소인덱스 + 마지막원소인덱스) / 2 이다.

| -- | ----------- | ----- | ------------- |
| 짝수 | `0 ~ n/2-1` | 없음    | `n/2 ~ n-1`   |
| 홀수 | `0 ~ n/2-1` | `n/2` | `n/2+1 ~ n-1` |

| -- | ------------- | ----- | --------------- |
| 짝수 | `0 until n/2` | 없음    | `n/2 until n`   |
| 홀수 | `0 until n/2` | `n/2` | `n/2+1 until n` |
