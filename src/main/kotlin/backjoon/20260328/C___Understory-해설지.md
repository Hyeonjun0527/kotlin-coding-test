# C - Understory 해설지

## 처음 어떤 신호를 보고 이 알고리즘을 떠올려야 했는가
연산이 `높이 추가`와 `x 이하 전부 제거` 두 가지뿐이고, 매번 필요한 출력이 "남은 개수"라는 점이 신호다. 개별 원소의 정확한 순서는 필요 없고, 가장 작은 값부터 지워질 수 있으면 되므로 최소 힙이 정답 구조다.

## 내가 왜 그걸 못 떠올렸는가
`조건을 만족하는 여러 원소를 한꺼번에 삭제`라는 문장을 보면 리스트를 직접 훑어서 지우는 그림이 먼저 떠오른다. 하지만 이 문제는 삭제 대상이 항상 "작은 값들 쪽"에 몰려 있어서, 정렬 상태를 유지하는 자료구조가 훨씬 자연스럽다.

## 틀린 방향은 왜 틀렸는가
배열에 다 넣어두고 매 삭제마다 전부 순회하면 연산마다 선형 시간이 든다. 삭제 조건이 `<= x` 하나뿐인데 정렬성을 안 쓰면, 문제의 구조를 버리고 힘으로만 푸는 셈이다.

## 다음에 비슷한 문제가 나오면 어디를 먼저 볼 것인가
삭제 조건이 `최솟값부터 연속 구간`처럼 보이면 힙을 먼저 본다. 그리고 출력이 개수/최솟값처럼 일부 정보만 필요하면, 전체 컨테이너를 정확히 유지하려 하지 말고 우선순위 큐로 줄일 수 있는지 본다.
